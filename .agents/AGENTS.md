<RULE[file_management]>
IMPORTANT FILE MANAGEMENT RULES

- Always modify existing project files whenever possible.
- Do NOT create duplicate, backup, copy, fixed, updated, new, v2, final, or alternative files.
- Before creating any new file, first check whether the required functionality can be implemented inside an existing file.
- Create a new file only when it is genuinely required by the architecture.
- If a new file is necessary, explain why it is needed.
- Never leave unused or orphaned files in the project.
- Maintain a clean file structure and avoid file duplication.
</RULE[file_management]>
